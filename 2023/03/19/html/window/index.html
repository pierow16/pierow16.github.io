window 以及 dom 以及 bom 对象的属性方法归纳 js执行机制

window

  • window对象是一个全局对象
  • window之下：navigator location document history screen
  • 函数 以及 通过var定义在全局作用域的变量都会变成window对象的属性和方法

BOM

JS执行机制

浏览器有两个引擎：渲染引擎(html/css) 和 js解释/解析器

同步任务

同步任务都在主线程上执行，形成一个执行栈

异步任务

js的异步都是通过回调函数实现的。
一般，异步任务有三种： (耗时的)

  1. 普通事件，click、resize等 （dom板块
  2. 资源加载，load、error等
  3. 定时器，setInterval、setTimeout等 （timer板块
    ajax(网络板块)
    异步任务相关 添加到任务队列（任务队列也叫消息队列.

js执行机制 ：

  1. 先执行执行栈中的同步任务
  2. 异步任务放入任务队列中
  3. 一旦执行栈中所有同步任务执行完毕，系统会按次序读取任务队列中的异步任务，于是被读取的异步任务结束等待状态进入执行栈开始执行。

—— 事件循环 ——
主线程不断的重复获得任务、执行任务、再获取任务、再执行，所以这种机制被称为事件循环（event loop）

简而言之：主线程把异步任务丢给浏览器处理，浏览器处理好后，把计算结果推到任务队列。主线程执行完后会不时去任务队列查看是由有已执行好的异步任务，有就调到执行栈执行。

浏览器可以多线程，所以异步api是浏览器的功能。异步任务都是提交给浏览器/宿主去执行的。

有异步任务时候。主线程会把异步任务丢给异步api去处理执行，异步api会把执行完毕(即处理好，已得到结果)的异步任务放到任务队列里.（即任务队列里存放的其实是执行完了的异步任务。

主线程执行完同步任务之后会不断查询任务队列里有没有任务，有就调过来执行。

js特点是单线程：同一时间只能做一件事

定时器 setInterval / setTimeout

每次调用定时器都会产生一个新的定时器，返回值是id （即12345 第几个定时器。

间歇函数：setInterval(回调函数，等待的毫秒数)

一直执行，除非手动清除

延时函数：setTimeout(回调函数，等待的毫秒数)

只执行一次

location对象

拆分并保存了url地址的各个组成部分

常用属性：

  • location.search

    获取地址栏中携带的参数，符号?后面部分（包括？
    可以拿到地址栏表单的相关信息!!!
    location.search // ?username=piero&age=18

  • location.hash

    获取地址中的哈希值，符号#后面部分(包含#)

    应用场景： spa：单页面应用。 网易云音乐就是 不需要刷新页面，加载资源少，只改变中间一块的内容。 根据hash地址即#后面地址显示不同内容，类似tab切换，但是组件的形式 比如网易云 'https://music.163.com/#/friend' 'https://music.163.com/#/my/'

  • location.href

    获取地址 赋值时候是跳转页面。
    使用场景：比如登陆成功后五秒钟之后自动跳转回首页/支付完几秒后跳转回之前页面。

常用方法

  • location.reload

    刷新当前页面，当传入参数true时表示强制刷新
    location.reload(true) //f5
    location.reload() //ctrl+f5
